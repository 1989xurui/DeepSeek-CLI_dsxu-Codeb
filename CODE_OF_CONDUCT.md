# Code of Conduct

DSXU Code uses a simple contributor standard:

- Be respectful and specific.
- Assume good intent, but keep technical claims evidence-based.
- Do not post secrets, private transcripts, or unredacted logs.
- Keep review comments focused on behavior, risk, tests, security, maintainability, and user experience.
- Do not use harassment, insults, or discriminatory language.

Maintainers may remove comments or contributions that make collaboration unsafe or that expose private credentials or user data.
